ClosedXML is a .NET library that lets you create, read, and modify Excel spreadsheets without needing Microsoft Excel installed. It wraps the low-level OpenXML SDK with an intuitive, fluent API that makes working with workbooks, worksheets, cells, tables, and formulas straightforward.

## What you can build

ClosedXML is used to generate Excel reports on web servers, automate data exports, and process uploaded spreadsheets — all without any dependency on the Excel application itself.

```csharp quickstart.cs
using ClosedXML.Excel;

using var workbook = new XLWorkbook();
var worksheet = workbook.AddWorksheet("Sales Report");

worksheet.Cell("A1").Value = "Product";
worksheet.Cell("B1").Value = "Units Sold";
worksheet.Cell("C1").Value = "Revenue";

worksheet.Cell("A2").Value = "Widget A";
worksheet.Cell("B2").Value = 1500;
worksheet.Cell("C2").Value = 45000.00;

// Apply bold formatting to the header row
worksheet.Row(1).Style.Font.Bold = true;

workbook.SaveAs("SalesReport.xlsx");
```

## Get started

<Steps>
  <Step title="Install via NuGet">
    Add ClosedXML to your .NET project using the .NET CLI or Package Manager Console.

    ```bash
    dotnet add package ClosedXML
    ```
  </Step>
  <Step title="Create a workbook">
    Instantiate `XLWorkbook`, add a worksheet, and set cell values.

    ```csharp
    using var workbook = new XLWorkbook();
    var ws = workbook.AddWorksheet("Sheet1");
    ws.Cell("A1").Value = "Hello, World!";
    ```
  </Step>
  <Step title="Save the file">
    Save to a file path or stream. The file is a standard `.xlsx` that opens in Excel, LibreOffice, and Google Sheets.

    ```csharp
    workbook.SaveAs("output.xlsx");
    ```
  </Step>
  <Step title="Read an existing file">
    Load an existing `.xlsx` file and read cell values directly.

    ```csharp
    using var workbook = new XLWorkbook("existing.xlsx");
    var ws = workbook.Worksheet("Sheet1");
    var value = ws.Cell("A1").GetValue<string>();
    ```
  </Step>
</Steps>

<Note>
  ClosedXML targets .NET Standard 2.0 and runs on .NET Core 2.0+, .NET Framework 4.6.2+, and Blazor. It is not thread-safe — do not share workbook instances across threads.
</Note>
